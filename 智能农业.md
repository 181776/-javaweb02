# 农作物产量预测与种植决策系统 — 需求文档

## 1. 项目背景与目标

### 1.1 背景
农业种植决策长期依赖经验，缺乏对历史产量、气象条件等数据的系统化分析。面对不同地区、不同作物、不同气候条件，种植者难以快速判断预期产量并做出合理种植安排。

### 1.2 目标
构建一个面向展示与决策支持的农业大数据智能分析应用，实现：

```
多源数据采集 → 批处理清洗 → 产量预测模型 → 可视化大屏 → 种植决策建议
```

### 1.3 交付形态
- 最终交付为 **Web 网页应用**，浏览器打开即可使用，无需安装客户端
- 不设用户登录，打开即用（生产环境可扩展权限管理）

---

## 2. 用户与场景

| 用户 | 典型场景 |
|------|----------|
| 农业从业者 | 查看某地区历史产量，输入气象条件获取预测结果 |
| 管理人员 | 通过数据大屏对比各地区、各作物产量趋势 |
| 课程演示观众 | 直接访问系统，查看预测效果与可视化分析 |

**核心使用流程：**
打开网页 → 查看历史数据 / 数据大屏 → 输入地区、作物、气象条件 → 获取预测产量与种植建议

---

## 3. 功能需求

### 3.1 数据管理
- 气象 / 土壤 / 产量数据查询（分页、按地区 / 年份 / 作物筛选）
- 支持 CSV 批量导入（可选，用于补充或更新数据）
- 展示数据来源与记录条数（体现多源融合）

### 3.2 智能分析
- **产量预测**：输入地区、作物、温度、降水等 → 输出预测产量
- **影响因素分析**（可选）：展示各特征对产量的影响权重
- **种植建议**：基于预测结果与历史数据的简单规则（如：某地区某作物预测产量高于历史均值则建议优先种植）

### 3.3 可视化大屏
- 产量趋势折线图（按年份）
- 地区 / 作物对比柱状图
- 预测 vs 实际对比（可选）
- 地图分布（可选，有地图数据后再做）

## 5. 数据需求

### 5.1 核心字段（建模与展示统一使用）

| 字段名 | 类型 | 说明 | 是否必需 |
|--------|------|------|----------|
| region | 字符串 | 地区 / 省份 | 是 |
| year | 整数 | 年份 | 是 |
| crop | 字符串 | 作物名称（如小麦、玉米、水稻） | 是 |
| temperature | 浮点数 | 年均温度（℃） | 是 |
| rainfall | 浮点数 | 年降水量（mm） | 是 |
| yield | 浮点数 | 产量（如 kg/亩 或 吨/公顷，需统一单位） | 是 |
| soil_ph | 浮点数 | 土壤 pH 值 | 否 |
| fertilizer | 浮点数 | 化肥用量 | 否 |

### 5.2 数据源策略（多源融合）

| 优先级 | 来源 | 说明 | 占比 |
|--------|------|------|------|
| 主数据源 | Kaggle 公开数据集 | 如 Crop Yield Prediction、Agriculture crop yield 等 | ~80% |
| 辅数据源 | 网络爬虫（可选） | 统计局等公开页面的分省产量、播种面积等 | ~20% |
| 合并目标 | 清洗后统一入库 | 字段对齐、单位统一、去重去空 | ≥ 1 万条 |

**数据采集原则：**
1. 优先使用公开 CSV 数据集，保证字段完整、数据量达标
2. 仅在主数据缺少某些维度时，再通过爬虫或第二份数据集补充
3. 不在项目初期依赖纯爬虫获取全部数据

### 5.3 数据量与质量要求
- 清洗后有效记录：**≥ 10,000 条**
- 至少覆盖：**3 种以上作物**、**5 个以上地区**、**5 年以上年份**
- 缺失值处理、异常值过滤、单位换算需在批处理脚本中完成并记录在报告中

### 5.4 数据目录规划

```
data/
├── raw/          # 原始数据（Kaggle 下载、爬虫结果）
├── clean/        # 清洗合并后的 CSV
└── scripts/      # 清洗、合并、入库脚本
```

## 6. 技术架构

### 6.1 技术选型

| 层次 | 技术 | 职责 |
|------|------|------|
| 前端展示 | Vue 3 + ECharts + Axios + Element Plus | 页面、大屏、调用后端 API |
| 后端业务 | Spring Boot + MyBatis + MySQL | REST API、数据 CRUD、转发预测请求 |
| 智能分析 | Python + FastAPI + pandas + scikit-learn + joblib | 批处理清洗、模型训练与推理 |
| 数据采集 | requests + BeautifulSoup（可选） | 辅助数据源爬取 |
| 缓存（可选） | Redis | 大屏热点统计数据 |

### 6.2 系统架构

```
浏览器（Vue + ECharts 网页）
        │ HTTP
        ▼
Spring Boot + MyBatis（端口 8080）
        │ HTTP              │ SQL
        ▼                   ▼
Python 分析服务（8000）   MySQL
  pandas / sklearn
        ▲
        │
  CSV 批处理脚本 ← raw 多源原始数据
```

### 6.3 大数据体现（课程报告要点）
- **多源数据**：Kaggle 数据集 + 可选网络采集
- **数据规模**：万级结构化记录
- **批处理**：Python pandas 清洗、合并、特征工程
- **智能分析**：scikit-learn 回归模型（如随机森林）进行产量预测
- **可扩展说明**：生产环境可接入 Spark 离线分析、Kafka 实时数据流、HDFS 分布式存储

### 6.4 项目目录规划

```
agri-bigdata/
├── agri-backend/       # Spring Boot 主服务
├── agri-analysis/      # Python 分析服务（FastAPI）
├── agri-frontend/      # Vue 前端
├── data/
│   ├── raw/
│   ├── clean/
│   └── scripts/
├── models/             # 训练好的 .pkl 模型

```

## 7. 接口概要

### 7.1 后端 API（Spring Boot → 前端）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/yield/list | 分页查询产量数据（支持 region、year、crop 筛选） |
| GET | /api/yield/{id} | 查询单条记录 |
| GET | /api/dashboard/stats | 大屏统计数据（总量、均值、Top 地区等） |
| GET | /api/dashboard/trend | 产量趋势数据 |
| POST | /api/predict | 发起产量预测（转发至 Python 服务） |
| GET | /api/advice | 获取种植建议（基于预测与历史规则） |

### 7.2 分析服务 API（Python → Spring Boot 内部调用）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /predict | 输入特征 → 返回预测产量 |
| GET | /health | 服务健康检查 |
| POST | /train | 可选，离线重新训练模型 |

**预测请求示例：**
```json
{
  "region": "河南",
  "crop": "小麦",
  "year": 2024,
  "temperature": 14.5,
  "rainfall": 580.0
}
```

**预测响应示例：**
```json
{
  "predicted_yield": 420.6,
  "unit": "kg/亩",
  "model": "RandomForestRegressor"
}
```

## 8. 数据库设计

- 数据库名：`smart_agriculture`
- 建表脚本：`smart agriculture/sql/init.sql`
- 详细说明：`smart agriculture/docs/database.md`
- 主数据来源：`yield_df.csv`（28,242 条，101 地区，10 作物，1990～2013）

### 8.1 产量事实表 `crop_yield`

| 字段 | 类型 | CSV 对应 | 说明 |
|------|------|----------|------|
| id | BIGINT | — | 主键 |
| region | VARCHAR(64) | Area | 地区/国家 |
| crop | VARCHAR(64) | Item | 作物 |
| year | INT | Year | 年份 |
| yield_value | DECIMAL(12,2) | hg/ha_yield | 产量（hg/ha，避免 yield 保留字） |
| rainfall | DECIMAL(10,2) | average_rain_fall_mm_per_year | 年均降水 mm |
| pesticides | DECIMAL(12,2) | pesticides_tonnes | 农药用量 tonnes |
| temperature | DECIMAL(5,2) | avg_temp | 年均温度 ℃ |
| source | VARCHAR(32) | — | 默认 kaggle |
| create_time | DATETIME | — | 入库时间 |
| update_time | DATETIME | — | 更新时间 |

**唯一索引：** `(region, crop, year)` — 入库时对重复数据去重

### 8.2 预测记录表 `predict_record`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| region | VARCHAR(64) | 输入地区 |
| crop | VARCHAR(64) | 输入作物 |
| year | INT | 输入年份（可空） |
| temperature | DECIMAL(5,2) | 输入温度 |
| rainfall | DECIMAL(10,2) | 输入降水量 |
| pesticides | DECIMAL(12,2) | 输入农药（可空） |
| predicted_yield | DECIMAL(12,2) | 预测产量 |
| model_name | VARCHAR(64) | 模型名称 |
| create_time | DATETIME | 预测时间 |

### 8.3 统计视图

| 视图 | 用途 |
|------|------|
| v_yield_summary | 明细查询 |
| v_region_stats | 按地区汇总（大屏） |
| v_crop_stats | 按作物汇总（大屏） |
